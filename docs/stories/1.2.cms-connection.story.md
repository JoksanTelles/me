# 📋 Story 1.2: CMS Connection & Configuration

**File:** `docs/stories/1.2.cms-connection.story.md`
**Status:** **Ready for Dev**

### 📖 User Story

**As a** Content Creator,
**I want** to connect the Nuxt application to Storyblok,
**so that** I can manage and retrieve content dynamically via the API without modifying the codebase.

### ✅ Acceptance Criteria

1.  Cuenta y "Space" de Storyblok creados y configurados.
2.  Módulo `@storyblok/nuxt` instalado en el proyecto.
3.  Token de acceso (Preview) configurado de forma segura en variables de entorno (`.env`).
4.  El archivo `nuxt.config.ts` inicializa el módulo de Storyblok correctamente.
5.  La página de inicio (`pages/index.vue`) obtiene datos reales desde Storyblok (ej. un título "Hello World").
6.  El "Visual Editor" de Storyblok funciona (puedes ver tu sitio local dentro del iframe de Storyblok - requiere HTTPS local o túnel, pero para MVP basta con que cargue datos).

### 🛠️ Dev Notes

> **Source Context:** `docs/architecture.md` (Tech Stack & Integrations)

  * **Storyblok Setup:**
      * Crear un Space nuevo.
      * Obtener el `PREVIEW_TOKEN` desde Settings \> Access Tokens.
  * **Nuxt Module:**
      * `npm install @storyblok/nuxt`
  * **Configuración:**
      * En `nuxt.config.ts`:
        ```typescript
        modules: [['@storyblok/nuxt', { accessToken: process.env.STORYBLOK_TOKEN }]]
        ```
      * **Importante:** No hardcodear el token. Usar `.env`.
  * **Componentes:**
      * Crear un componente base `StoryblokComponent.vue` (o usar el default del módulo) para renderizar los bloques dinámicamente.
      * El componente `Page.vue` (en `components/storyblok/Page.vue`) debe mapear al content type "Page" de Storyblok.
