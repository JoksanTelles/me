# 📋 Story 1.1: Project Setup & Initialization

**File:** `docs/stories/1.1.project-setup.story.md`
**Status:** **Ready for Dev**

### 📖 User Story
**As a** Developer,
**I want** to initialize the Nuxt 3 project with Tailwind, Shadcn-vue, and the Cloudflare configuration,
**so that** I have a solid, deployable codebase foundation to start building features.

### ✅ Acceptance Criteria
1.  Repositorio Git inicializado con estructura limpia.
2.  Nuxt 3 instalado y ejecutándose correctamente en modo desarrollo (`npm run dev`).
3.  Tailwind CSS configurado y funcionando.
4.  **Shadcn-vue** instalado e inicializado correctamente (estructura de componentes visible).
5.  Configuración de despliegue para **Cloudflare Pages** establecida (preset Nitro).
6.  El proyecto compila sin errores (`npm run build`).

### 🛠️ Dev Notes
> **Source Context:** `docs/architecture.md` (Tech Stack & Project Structure)

* **Framework:** Utilizar `npx nuxi init` para la última versión de Nuxt 3.
* **Language:** TypeScript es estricto por defecto.
* **UI Library:** Instalar `shadcn-vue` siguiendo la guía oficial para Nuxt.
    * CLI: `npx shadcn-vue@latest init`
    * Configuración: Usar `Slate` o `Zinc` como color base según `front-end-spec.md`.
* **Hosting:** Configurar el preset de Nitro para Cloudflare Pages en `nuxt.config.ts`.
* **Package Manager:** Usar `npm` o `pnpm` (ser consistente).
* **Estructura de Carpetas:** Asegurar que se creen las carpetas base definidas en la arquitectura: `components/ui`, `layouts`, `pages`, `server/api`.

### 📝 Tasks
- [ ] Inicializar proyecto Nuxt 3 (`npx nuxi init joksan-dev`).
- [ ] Inicializar repositorio Git y crear `.gitignore`.
- [ ] Instalar dependencias de Tailwind CSS (vía módulo `@nuxtjs/tailwindcss` o manual según prefiera Shadcn).
- [ ] Ejecutar CLI de Shadcn-vue para configurar `components.json` y estilos base.
- [ ] Instalar componente de prueba (ej. `Button`) para verificar Shadcn: `npx shadcn-vue@latest add button`.
- [ ] Configurar `nuxt.config.ts` con:
    -   `nitro: { preset: 'cloudflare-pages' }`
    -   Módulos necesarios (aún no instalar Storyblok/i18n, solo base UI).
- [ ] Limpiar `app.vue` y crear página de prueba `pages/index.vue` con un botón Shadcn.
- [ ] Ejecutar build local y verificar carpeta `.output`.
